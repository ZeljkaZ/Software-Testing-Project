*** Settings ***

Documentation  Testing the Project page in GitLab. This test suite includes test cases that cover CRUD operations 
(Creating, Reading, Updating and Deleting projects). Both positive and negative scenarios are included to check that
projects are created successfully and that the system shows error messages when invalid data is used.


Pre Condition  User is registered and logged in
Post Condition	User logs out and closes browser
Tags  Project, TS-3-Project, CRUD

#******************** Test Cases *************************

*** Test Cases ***

# Positive test cases

Create Blank Project
    Navigate To Project Page  Project_Page_Url
    Click New Project
	Choose Create Blank Project
    Input Project Name   Valid_Project_Name
	Pick a Group or Namespace   Project_Group
	Input Project Slug (Optional)   Project_Slug  # This step can be skipped as project name is automatically added as a project slug
	Select Deployment Target (Optional)
	Select Visibility Level (Optional)
	Complete Project Configuration (Optional)
	Toggle Experimental Settings (Optional)
    Click Create Project Button
    Verify Page Should Contain  "Project 'Valid_Project_Name' was successfully created."


Create Template Project
    Navigate To Project Page  Project_Page_Url
    Click New Project
	Choose Create From Template
	Choose One Of The Templates
	Click Use Template
    Input Project Name   Template_Project_Name
	Pick a Group or Namespace  Project_Group
	Pick a Project Slug  Alternative_Project_Slug  # Optional step, as system automatically adds project slug
	Add Project Description (Optional)
	Select Deployment Target (Optional)
	Select Visibility Level (Optional)
    Click Create Project Button
	Wait For Importing of Project To Be Complete
    Verify Page Should Contain   "The Project Was Successfully Imported"


Update Name Of Template Project
    Navigate To Project Page  Project_Page_Url
	Click Members
    Select Project   Template_Project_Name
	Choose 3 Dots In The Upper Right Corner
	Select Project Settings
    Change Project Name   Updated_Project_Name
    Click Save Changes Button
    Verify Page Should Contain   "Project 'Updated_Project_Name' was successfully updated."


Delete Template Project
    Navigate To Project Page  Project_Page_Url
	Click Members
    Select Project   Template_Project_Name
	Choose 3 Dots In The Upper Right Corner
	Select Project Settings
	Select Advanced
    Click Delete Button
	Enter Suggested Info To Confirm (e.g. zz-group3/my-template-project or similar path)
    Confirm Deletion
    Verify Page Should Contain Project Deletion Schedule


Create Project With Used Project Name but Different Namespace (Group)
    Navigate To Project Page  Project_Page_Url
    Click New Project
	Choose Create Blank Project
    Input Project Name   Duplicate_Project_Name
	Pick a Group or Namespace   Alternative_Project_Group
	Input Project Slug   Project_Slug  # Optional step as project slug is automatically entered but can be changed manually
	Select Deployment Target (Optional)
	Select Visibility Level (Optional)
	Complete Project Configuration (Optional)
	Toggle Experimental Settings (Optional)
    Click Create Project Button
    Verify Page Should Contain   "Project 'My Project' was successfully created."
